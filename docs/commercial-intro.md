# VoStory — AI 有声内容智能制作平台

## 一句话介绍

VoStory 是一个面向有声内容制作团队的 AI 生产平台，将长篇小说自动转化为结构化广播剧脚本，实现角色智能识别、音色自动适配、批量语音生成与团队协作交付的全流程覆盖。

## 行业痛点

有声小说和广播剧市场持续增长，但制作流程仍然高度依赖人工，存在以下核心问题：

### 1. 脚本制作效率极低

一本 50 万字的网络小说，人工拆分章节、标注对白旁白、识别说话人、标记情绪，通常需要 **数周甚至数月**。制作团队大量时间消耗在重复性的文本整理工作上，而非创作本身。

### 2. 角色管理混乱

长篇小说中角色众多，同一角色可能有正名、别名、称呼、代词等多种表达方式。人工整理角色表容易遗漏或重复，导致后续配音环节出现声音错配。

### 3. 配音成本居高不下

传统方式依赖真人配音演员，一本书的配音成本从数千到数万元不等。即使使用 AI 语音合成，也面临角色音色不统一、情绪表达生硬、逐句手动调参等问题。

### 4. 返工代价巨大

发现某个角色的声音不合适、某段台词的情绪不对，往往需要整章甚至整本重新生成。缺乏片段级的精细控制能力，导致返工成本极高。

### 5. 团队协作困难

多数工具是单机版桌面软件，无法支持多人分工。编辑改脚本、导演调声音、制作人审核导出，这些环节无法在同一个平台上协同完成。

### 6. 资产无法复用

每个项目的角色配置、声音方案、发音词典都是一次性的。做完一本书，下一本书一切从零开始，积累的经验和资产无法沉淀。

## VoStory 如何解决

### 智能章节切分

导入一本完整小说（支持 txt / docx / epub），平台自动完成章节识别与切分。长篇内容按可配置字数上限分批处理，每批独立调用 LLM，单批失败不影响整体，支持断点续跑。

**效果**：50 万字小说的章节切分从人工数天缩短到分钟级自动完成。

### 自动角色抽取

基于 LLM 智能分析全文，自动识别所有角色并归并别名与称呼。系统区分主角、配角、路人三个层级，输出结构化的角色候选列表，人工只需审核和微调。

**效果**：角色整理工作量减少 80% 以上，避免遗漏和重复。

### 脚本自动结构化

自动将小说段落拆分为脚本片段，识别每个片段的类型（旁白 / 对白 / 内心独白），归属说话人，并标注情绪类型与情绪强度。输出的结构化脚本可直接用于后续配音生产。

**效果**：从原文到可编辑脚本的转化全自动完成，人工聚焦于审核修正而非从零标注。

### 智能音色适配

平台提供全局声音资产库，支持为每个角色绑定专属音色。同一角色可按不同情绪（开心、悲伤、愤怒、恐惧等）绑定不同的参考音频，TTS 引擎根据脚本中的情绪标签自动选择对应音色生成语音。

**效果**：角色声音在整本书中保持一致，情绪表达自然丰富，无需逐句手动调参。

### 批量生成与局部返工

支持按章节一键批量生成全部语音，通过 SSE 实时推送进度。更关键的是，支持**片段级重生成**——只对有问题的某一句话重新生成，无需整章重跑。

**效果**：批量生成节省 90% 以上的手动操作，局部返工成本从"整章重来"降低到"改一句话"。

### 团队协作与权限管理

基于工作区（Workspace）的团队管理，支持 RBAC 权限控制。编辑负责脚本修正，导演负责声音调配，制作人负责审核导出，各司其职，在同一个平台上协同工作。

**效果**：告别文件传来传去的低效协作，全流程在线化、可追溯。

### 资产沉淀与跨项目复用

声音资产分为全局资产库（VoiceAsset）和项目内配置（VoiceProfile）两层。做完一本书积累的角色音色、发音词典、Prompt 模板，可以直接复用到下一本书。

**效果**：团队的制作经验和资产持续积累，项目越做越快。

## 适用场景

### 有声小说批量制作

网文平台、阅读 App 需要将海量小说转化为有声内容。VoStory 支持批量导入、自动化处理、团队分工，适合规模化生产。

### 广播剧制作

广播剧工作室需要精细的角色管理、情绪控制和多轨音频处理。VoStory 的角色资产体系和情绪双维度系统为广播剧制作提供专业级支持。

### 内容平台有声化

内容平台希望将文字内容快速转化为音频形态，拓展分发渠道。VoStory 提供从导入到导出的完整自动化链路，大幅降低有声化门槛。

### 个人创作者

独立作者希望将自己的作品制作成有声版本，但缺乏专业团队和预算。VoStory 让一个人也能完成从小说到有声书的全流程制作。

## AI 服务配置

### LLM 提供商

VoStory 采用 Provider 策略模式，LLM 支持多厂商灵活切换：

- OpenAI / Azure OpenAI
- DeepSeek
- Anthropic Claude
- Google Gemini
- Ollama（本地部署，数据不出境）
- 阿里云百炼
- 任意 OpenAI 兼容接口

所有 LLM 配置存储在数据库中，运行时切换，无需重启服务。

### TTS 引擎

VoStory 的语音合成基于 **IndexTTS2**，这是目前最适合有声内容制作的开源 TTS 引擎：

- **零样本声音克隆**：只需一段参考音频即可克隆音色，完美匹配角色声音配置
- **情绪与音色解耦**：独立控制音色和情绪，同一角色可在不同情绪下保持音色一致
- **多模态情绪控制**：支持情绪参考音频、8 维情绪向量、文本情绪描述三种方式
- **情绪强度控制**：0.0-1.0 连续值，精确控制情绪表达程度
- **拼音标注**：原生支持中文混合拼音，直接对接发音词典

VoStory 提供 IndexTTS2 的 **Docker 镜像**，用户自行部署 TTS 服务，在平台中配置服务地址即可使用。数据完全本地化，不依赖任何第三方云服务。

## 数据安全与合规

- **私有化部署**：全套系统可部署在客户自有服务器，数据不出境
- **本地 AI 可选**：支持 Ollama 本地 LLM，TTS 基于 IndexTTS2 本地部署，语音数据不出境
- **权限隔离**：工作区级别的数据隔离，RBAC 细粒度权限控制
- **操作审计**：完整的操作日志和 LLM 调用日志，满足合规审计需求
- **版权预留**：系统设计预留了小说版权授权、声音克隆授权等合规管理能力
