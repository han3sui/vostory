# 多音轨编辑器技术方案

## 一、需求分析

Vostory 的有声书制作流程中，TTS 合成完成后需要一个**多音轨编辑页面**，用于：

- **多轨道时间线**：按角色/片段类型分轨道，可视化排列音频片段
- **波形显示**：每个音频片段展示波形预览
- **拖拽编排**：调整片段在时间轴上的位置、顺序、间隔
- **播放控制**：多轨道混合播放、单轨 Solo/Mute
- **片段操作**：裁切、淡入淡出、插入静音间隔
- **导出合成**：将多轨道合成为最终音频文件

## 二、方案对比

### 方案 1：WaveSurfer.js + 自定义时间线

| 维度       | 评价                                           |
| ---------- | ---------------------------------------------- |
| 地址       | https://wavesurfer-js.org/                     |
| 成熟度     | 非常成熟，社区活跃，GitHub Star 9k+            |
| Vue 3 兼容 | 纯 JS 库，封装成 composable 即可               |
| 多轨道     | 官方 Regions 插件 + 多实例叠加                 |
| 波形渲染   | 内置，Canvas 渲染，性能好                      |
| 学习成本   | 低                                             |
| 定制灵活度 | 高（UI 完全自定义）                            |

**核心思路**：
- 每个音频片段用一个 WaveSurfer 实例渲染波形
- 用 Vue 3 组件 + CSS Grid 做轨道布局（按角色分轨道）
- 播放控制用 Web Audio API 做多轨同步
- 拖拽用 `vue-draggable-plus` 或自定义实现

**优点**：
- 波形渲染质量高，API 简洁
- 完全掌控 UI/UX，可以和现有设计语言统一
- 按需引入，包体小

**缺点**：
- 多轨混合播放、复杂拖拽编排需要自己实现
- 不是开箱即用的 DAW 体验，需要组装

---

### 方案 2：waveform-playlist

| 维度       | 评价                                           |
| ---------- | ---------------------------------------------- |
| 地址       | https://github.com/naomiaro/waveform-playlist  |
| 定位       | 专门做多轨道音频编辑器的开源库                 |
| 功能       | 多轨道、波形、拖拽、裁切、淡入淡出、混合播放   |
| Vue 3 兼容 | 原生 JS，需要手动集成到 Vue 组件               |
| 社区       | GitHub Star 1.5k+，功能完整但社区较小          |
| 学习成本   | 中                                             |

**核心思路**：
- 直接初始化 waveform-playlist 实例，传入轨道配置
- 将项目音频数据适配为 playlist 的数据格式
- 自定义工具栏和操作按钮

**优点**：
- 最接近"开箱即用的多轨编辑器"
- 多轨同步播放、波形渲染、拖拽排列全内置
- 支持淡入淡出、裁切等编辑操作

**缺点**：
- UI 定制灵活度一般，深度定制需要改源码
- 社区较小，遇到问题支持少
- 样式不易与 Arco Design 统一

---

### 方案 3：Tone.js + Canvas 自绘时间线

| 维度       | 评价                                           |
| ---------- | ---------------------------------------------- |
| 地址       | https://tonejs.github.io/                      |
| 音频能力   | 专业级，基于 Web Audio API 的高层封装          |
| 多轨同步   | 内置 Transport、精确调度、多音源同步播放       |
| 效果处理   | 支持混响、EQ、压缩、淡入淡出等                |
| UI         | 无 UI，时间线/波形/拖拽全部自绘               |
| 学习成本   | 中高                                           |

**核心思路**：
- Tone.js 负责音频播放层（Transport 同步、效果链）
- Canvas/SVG 自绘时间轴、波形、轨道
- Vue 3 组件管理交互状态

**优点**：
- 音频同步播放精度最高
- 支持复杂音频效果处理
- 完全自由的 UI

**缺点**：
- UI 工作量巨大（时间轴、波形绘制、拖拽全部手写）
- 波形解码 + Canvas 渲染需要额外开发
- 开发周期长

---

### 方案 4：Peaks.js（BBC 开源）

| 维度       | 评价                                           |
| ---------- | ---------------------------------------------- |
| 地址       | https://github.com/bbc/peaks.js                |
| 背景       | BBC 开源，专为长音频浏览设计                   |
| 波形       | 支持服务端预生成波形数据，长音频性能优         |
| 多轨道     | 不直接支持，需要多实例                         |
| 学习成本   | 中                                             |

**适合场景**：单章节长音频的精确浏览和标记。

**不推荐原因**：设计上不面向多轨道编排，强行做多轨需要大量额外开发。

| 维度   | 评价       |
| ------ | ---------- |
| 灵活度 | 最高       |
| 工作量 | 巨大       |
| 周期   | 数周到数月 |

仅在团队有专业音频前端开发经验、且产品形态高度定制时考虑。

## 四、推荐路线（分阶段）

### 第一阶段：WaveSurfer.js + 自定义轨道布局（MVP）

快速出效果，贴合现有架构。

**技术栈**：
- `wavesurfer.js` — 波形渲染
- `vue-draggable-plus` — 拖拽排序
- Web Audio API — 多轨同步播放（基础版）
- Vue 3 Composable — 封装音频实例管理

**页面结构**：
```
┌──────────────────────────────────────────────┐
│ 工具栏：播放/暂停 | 缩放 | 导出              │
├──────────────────────────────────────────────┤
│ 时间刻度尺                                    │
├────────┬─────────────────────────────────────┤
│ 旁白   │ [==波形==] [==波形==]    [==波形==]  │
├────────┼─────────────────────────────────────┤
│ 角色A  │    [==波形==]     [==波形==]         │
├────────┼─────────────────────────────────────┤
│ 角色B  │ [==波形==]           [==波形==]      │
├────────┼─────────────────────────────────────┤
│ 角色C  │         [==波形==]                   │
└────────┴─────────────────────────────────────┘
```

**Vue 3 集成示例**：
```typescript
import WaveSurfer from 'wavesurfer.js'

function useWaveform(containerRef: Ref<HTMLElement | null>, audioUrl: string) {
  const ws = shallowRef<WaveSurfer | null>(null)
  const duration = ref(0)
  const isPlaying = ref(false)

  onMounted(() => {
    if (!containerRef.value) return
    ws.value = WaveSurfer.create({
      container: containerRef.value,
      waveColor: '#4F4A85',
      progressColor: '#383351',
      url: audioUrl,
      height: 48,
      barWidth: 2,
      barGap: 1,
    })
    ws.value.on('ready', () => { duration.value = ws.value!.getDuration() })
    ws.value.on('play', () => { isPlaying.value = true })
    ws.value.on('pause', () => { isPlaying.value = false })
  })

  onBeforeUnmount(() => ws.value?.destroy())

  return { ws, duration, isPlaying }
}
```

**注意事项**：
- 用 `shallowRef` 而非 `ref`（避免 Vue 深层代理音频对象导致性能问题）
- `onBeforeUnmount` 必须 `destroy()`（释放 AudioContext）
- 大量片段时考虑虚拟滚动（只渲染可见区域的波形实例）
- AudioContext 有浏览器实例数限制，需要共享或按需创建

### 第二阶段：引入 Tone.js 增强播放层

在第一阶段基础上，替换播放控制层：

- Tone.js Transport 实现精确多轨同步
- 支持片段间隔（silence gap）的精确控制
- 支持交叉淡入淡出
- 保留 WaveSurfer 做波形渲染

### 第三阶段：服务端合成导出

- 前端编排好轨道数据（片段顺序、间隔、音量、淡入淡出参数）
- 提交给后端，由 FFmpeg 或类似工具做最终混音导出
- 前端不做实际的音频合成（避免浏览器内存/性能瓶颈）

## 五、与现有架构的对接

当前 Vostory 已有的数据结构可以直接映射：

| 现有数据              | 多轨编辑器映射        |
| --------------------- | --------------------- |
| `vs_character`        | 轨道（按角色分轨）    |
| `vs_script_segment`   | 片段（时间轴上的块）  |
| `vs_audio_clip`       | 片段关联的音频文件    |
| `segment.segment_num` | 片段在轨道内的排序    |
| `segment.character_id`| 片段所属轨道          |

不需要新建核心数据模型，只需新增一个"编排配置"（片段间隔、音量调整等参数）即可。
